# [프로젝트 회고] Vinyl Remote - 안드로이드 턴테이블 UI로 음악 앱 원격 제어하기

## 한 줄 소개
Spotify/YouTube Music의 재생 세션을 읽어서, 실제 턴테이블처럼 바늘을 움직여 재생/일시정지/탐색을 제어하는 Android 앱을 만들었습니다.

## 왜 만들었나
기존 음악 플레이어 UI는 기능 중심이지만, 감성적인 조작감은 부족하다고 느꼈습니다.  
그래서 "시각적 몰입감 + 실제 조작감"을 목표로 턴테이블 인터페이스를 구현했습니다.

## 핵심 구현 포인트

### 1) 바늘(톤암) 제스처 기반 재생 제어
- 바늘을 LP 위로 드래그하면 재생
- 바늘을 바깥(화이트 영역)으로 빼면 일시정지
- 슬라이더 탐색 시 바늘 위치가 실시간으로 함께 이동

### 2) 실제감 강화
- 플래터 회전 애니메이션 + 재생 위치 기반 톤암 이동
- 톤암 3레이어 메탈 라이팅(armBase/Shadow/Highlight)
- 스크래치 SFX, 바늘 드롭 햅틱

### 3) 가사 UX
- LRC 정식 파싱(타임태그 + offset 반영)
- 줄 단위 활성 가사 하이라이트
- 비동기 가사 탐색 및 수동 편집/저장

### 4) 디바이스 대응
- 폰/태블릿/가로모드 대응
- Foldable(Flip) 커버/오픈 상태별 프리셋 분기
- 극단적 화면비에서 컨트롤 우선순위 재배치

### 5) 외부 제어
- 알림/잠금화면에서 prev/play-next/needle in-out
- 홈 위젯에서도 동일 액션 지원

## 기술 스택
- Kotlin, Jetpack Compose, Material3
- MediaSession / NotificationListenerService
- StateFlow + Coroutines
- Gradle Kotlin DSL

## 시행착오
- 음악 앱마다 MediaSession metadata 품질이 달라서, 가사/탐색/duration 처리 분기가 필요했습니다.
- 폴더블에서 화면비만으로는 UX가 깨지기 쉬워 별도 프리셋을 분리했습니다.
- 바늘 떨림/반응성은 "너무 사실적"이면 오히려 불편해져, 시각효과와 조작성을 타협했습니다.

## 결과
- 단순 플레이어 리모컨이 아니라, "손맛" 있는 인터랙션 앱으로 완성
- 포트폴리오에서 UI/UX 감각 + Android 시스템 연동 능력을 동시에 보여줄 수 있는 프로젝트가 되었습니다.

## 앞으로 개선할 점
- 테마별 고급 쉐이더/라이팅 파라미터 분리
- 위젯 커스터마이징(버튼 배치/크기/투명도)
- 가사 싱크 수동 보정 UI(+/- ms)
- 릴리즈 빌드/서명/스토어 배포 준비 자동화
